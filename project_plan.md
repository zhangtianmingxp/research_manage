# 研究论文实验设计与公开数据溯源整理项目计划

## 一、项目名称

同步化与扰动相关研究论文的实验设计、样本时间轴及公共数据编号可核验整理

## 二、研究背景与意义

当前仓库包含 36 篇 PDF，其中 `文献/研究/` 下有 29 篇研究性论文，`文献/综述/` 下有 7 篇综述。研究论文涉及细胞周期同步化、染色质三维结构、SMC 复合物及相关扰动实验，但论文正文、补充材料和公共数据库对实验设计的描述通常分散在不同位置。项目目标是把研究性论文产生的数据、实验条件、同步化时间轴、扰动设计和公共归档编号连接成一套统一、可筛选、可追溯且便于人工核验的结构化结果。

本项目不是对论文结论做自由概述，而是建立“每个字段都能回到原始证据”的数据目录。任何无法由论文、补充材料或权威数据库确认的信息都必须明确标记为缺失、未报告、存在冲突或待核验，禁止凭常识、文件名、相似研究或数据库编号模式进行猜测。

## 三、核心科学问题

1. 每篇研究论文实际自产了哪些数据，哪些数据仅为复用、再分析或引用外部研究？
2. 每个实验的物种、样本、实验手段、同步化策略、同步化起点、采样时间点及对应细胞周期阶段是什么？
3. 每项扰动作用于什么生物对象，采用什么技术，直接靶标是什么，作者预期的作用及实际验证结果是什么？
4. 每个实验、样本和测序运行如何映射到 DOI、ENA/EMBL-EBI 项目、BioProject、SRA/ENA Run、GEO、BioSample 等公共编号？
5. ENA/EMBL-EBI 提供的项目页面和下载链接是否真实可访问，运行号是否确实隶属于所记录的项目和样本？
6. 在字段口径统一后，如何让所有论文的实验设计既能横向比较，又不丢失论文原始命名和证据上下文？

## 四、项目范围与关键定义

### 4.1 纳入范围

- 主分析对象：`文献/研究/` 下现有 29 篇研究性论文 PDF。
- 辅助材料：论文补充材料、期刊正文页面、数据可用性声明及权威公共数据库记录。
- `文献/综述/` 下 7 篇综述仅可用于发现线索和理解术语，不作为实验事实或数据编号的最终证据来源，也不进入研究论文主表。
- 若发现重复 PDF、预印本与正式发表版本或同一研究的不同文件，按 DOI、标题和作者核对后建立重复关系，不直接删除原文件或重复计数实验。

### 4.2 “本文自产数据”的判定

只有在论文正文、方法、补充材料、数据可用性声明或数据库提交者信息能够支持“该研究生成或委托生成该数据”时，才标记为 `yes`。复用既有数据标记为 `no_reused`；由自产数据计算得到的二次产物标记为 `derived_from_own`；证据不足标记为 `unclear`。不得仅因数据编号出现在论文中就判定为自产。

### 4.3 数据粒度

内部采用规范化多表结构；最终用户宽表原则上每行表示一个唯一的“论文 × 实验 × 生物样本 × 同步化/时间点 × 扰动条件 × 实验手段 × 数据对象或 Run accession”组合。同一样本对应多个 SRR/ERR/DRR 时逐运行展开。无测序运行号的显微、蛋白质组、成像、流式或其他数据也必须保留，并记录其对应归档库编号或“未公开/未报告”的证据状态。

### 4.4 缺失值规范

- `NR`：论文或补充材料未报告。
- `NA`：该字段对当前实验不适用。
- `NOT_FOUND`：已按既定来源检索但未找到。
- `UNRESOLVED`：来源之间存在冲突，暂不能裁决。
- `RESTRICTED`：数据受控访问、尚未公开或链接受限制。

空字符串不得代替上述状态；不得把未知值补成看似合理的答案。

## 五、数据来源与证据优先级

### 5.1 数据来源

1. 本地论文 PDF 正文及其中的数据可用性声明。
2. 同一论文的正式补充材料、方法附录和样本表。
3. 期刊官网的正式论文页面及补充文件。
4. EMBL-EBI/ENA 的 Study、Project、Experiment、Sample、Run 和 File records，以及 ENA Portal API 返回记录。
5. NCBI SRA、BioProject、BioSample、GEO；必要时使用 ArrayExpress/BioStudies、PRIDE、DDBJ 等与实验类型对应的权威归档库。
6. Crossref、PubMed 等权威书目信息源，用于 DOI、题名、年份等元数据核验。

搜索引擎结果、综述、聚合网站和二手数据库只能作为定位线索，不能单独支撑最终字段。

### 5.2 证据优先级

实验条件优先采用论文方法和补充材料；编号隶属关系、文件名和下载地址优先采用归档数据库当前记录。若来源冲突，保留各自原值、来源和检索日期，在 `unresolved_issues` 中登记，不静默覆盖。

每个关键事实至少记录：来源类型、来源定位、URL 或本地文件、页码/章节/表格/图号、数据库查询日期和核验状态。摘录只保留支持字段所需的最短上下文。

## 六、统一数据模型与字段要求

### 6.1 论文表 `papers`

- `paper_id`、本地 PDF 路径、规范题名、作者、期刊、年份。
- DOI、PMID（如有）、期刊页面 URL。
- 研究论文/综述分类、重复或版本关系。
- 数据可用性声明原始位置。
- 论文处理状态、双重核验状态、总体备注。

### 6.2 实验表 `experiments`

- `experiment_id`、`paper_id`、实验目的和实验组/对照组。
- 是否本文自产数据及判定证据。
- 实验手段、assay 类型、关键建库/检测技术、测量对象、分辨率或数据模态。
- 生物学靶点与检测靶点；例如抗体靶点、拉下靶点、测序或成像关注对象，避免与扰动靶标混淆。
- 生物学重复、技术重复、批次信息、配对关系和作者原始样本组名。
- 参考基因组、注释版本和其他影响复现的关键条件（若报告）。

### 6.3 生物样本与同步化表 `samples_timepoints`

- 物种学名、常用名、NCBI Taxonomy ID（可核验时）。
- 细胞系/原代样本/组织/菌株、样本名称原文、标准化名称。
- 基因型、性别、组织、发育阶段、培养条件等已报告信息。
- 同步化技术、同步化试剂或操作、浓度/剂量、处理时长、阻断点。
- 同步化起点或时间零点的精确定义，例如药物加入、阻断完成、洗脱释放、分选完成；若论文没有定义则记 `NR`。
- 释放后或处理后采样时间、单位、时间范围、是否 pooled。
- 采样点对应的细胞周期阶段、阶段判定证据和判定类型：`author_stated`、`marker_supported`、`explicitly_inferred` 或 `unknown`。
- 不得仅凭时间长度推断细胞周期阶段；确需根据作者给出的明确映射推导时，必须同时保存推导规则和原始时间点。

### 6.4 扰动表 `perturbations`

- 扰动对象：物种、细胞、蛋白、基因、复合物、染色质位点或其他实验对象。
- 扰动类型：降解、敲除、敲低、过表达、突变、抑制剂、招募、阻断、洗脱、温敏处理等。
- 扰动技术及具体系统，例如 CRISPR、RNAi、AID/dTAG、药理处理或工程化蛋白系统。
- 直接靶标、构建体/试剂、剂量、持续时间、处理相对同步化的时间关系。
- 阴性/阳性/载体/未处理对照。
- 作者预期效果，必须注明是论文声明还是根据实验设计的明确目标。
- 实际验证指标与观察结果；“预期效果”和“观测效果”分列，禁止混写。
- 若一个实验包含多个扰动，按扰动组合保留组合 ID，并在明细表逐项展开。

### 6.5 公共归档与下载表 `accessions`

- 数据库/命名空间：ENA、SRA、GEO、BioProject、BioSample、ArrayExpress/BioStudies、PRIDE、DDBJ 等。
- Project/BioProject accession，例如 `PRJEB`、`PRJNA`、`PRJDB`。
- ENA Study accession，例如 `ERP`，以及数据库实际提供的其他 study/series 编号。
- GEO Series/Sample、SRA Study、Experiment、Sample 和 Run accession。
- 逐项列出 `SRR`、`ERR` 或 `DRR`，不得把非 Run 编号伪装成 SRR。
- 作者样本名、数据库 sample alias、BioSample、实验和 Run 的对应关系。
- EMBL-EBI/ENA 官方项目/Study/Run 页面 URL。
- ENA API 实际返回的 FASTQ/提交文件下载链接、文件格式、文件大小、MD5（若数据库提供）。不得仅按编号模式手工拼接并视为已验证链接。
- 数据公开状态、受控访问状态、查询日期、链接检查状态。
- 论文声明的 accession 与数据库实际解析结果；不一致时登记冲突。

### 6.6 证据表 `evidence`

- `evidence_id`、所支持的表名/记录/字段。
- 来源类型、完整引文或数据库记录、URL/本地路径。
- PDF 页码、章节、补充表、图号或数据库查询条件。
- 最短必要证据摘录、检索日期、提取者、核验者。
- 证据等级：`primary_paper`、`supplement`、`archive_record`、`secondary_locator`。

### 6.7 问题表 `unresolved_issues`

- 缺失编号、归档记录不公开、链接失效、样本映射不唯一、正文与数据库冲突等问题。
- 已检查来源、检索式或 API 条件、当前判断、需要人工确认的具体问题。
- 问题状态和解决记录，确保“没有找到”也可审计。

## 七、方法路线与阶段划分

### 阶段 1：论文清单、去重与数据字典

1. 为 29 篇研究论文建立稳定 `paper_id` 和处理清单。
2. 从题名、DOI、作者、年份识别重复或版本关系。
3. 固化上述表结构、字段类型、受控词表、缺失值和唯一键。
4. 建立每篇论文的完成状态与检查清单。

### 阶段 2：逐篇提取论文级与实验级设计

1. 定位摘要、方法、数据可用性声明、补充材料入口和关键实验表。
2. 逐实验记录物种、样本、assay、同步化、时间点、细胞周期阶段和扰动设计。
3. 区分自产、衍生和复用数据；为每项判定附证据。
4. 所有原始名称与标准化名称并存，标准化不得覆盖原文。

### 阶段 3：公共编号展开与 ENA/EMBL-EBI 核验

1. 收集论文声称的所有 accession，并识别其命名空间。
2. 从 Project/Study 向 Sample、Experiment、Run 和文件逐层展开。
3. 交叉检查 ENA 与 NCBI/DDBJ 的镜像和映射，但保留原始归档命名空间。
4. 验证每个 Run 是否属于对应项目、实验和样本；核对运行总数，避免漏列。
5. 保存 ENA 官方页面和 API 返回的下载字段，检查链接状态并记录检查日期。

### 阶段 4：样本—时间点—扰动—数据运行映射

1. 使用论文样本表、数据库 alias 和建库信息建立映射。
2. 把同步化起点、处理时间、释放时间和采样时间统一到清晰时间轴，同时保留原始表述。
3. 将作者明确给出的细胞周期阶段与时间点关联；推导值单独标识。
4. 多因素组合实验拆分为可筛选字段，并保留组合标签和对照关系。
5. 无法唯一映射的记录不强行合并，进入问题表。

### 阶段 5：质量控制与逐篇人工核验

1. 对 DOI、Project、Study、Sample、Experiment、Run 编号进行格式和在线存在性检查。
2. 对“论文声明的 Run 数—数据库返回 Run 数—最终表 Run 数”做数量对账。
3. 检查主外键、唯一键、重复行、缺失率、受控词表和非法空值。
4. 抽查下载链接、项目隶属关系、自产数据判定、同步化起点和扰动靶标。
5. 每篇论文生成简短核验摘要，列出已确认内容、缺失信息、冲突及待人工问题。

### 阶段 6：汇总与用户友好交付

1. 生成规范化 TSV/CSV 表和一张用户友好的总宽表。
2. 总表支持按论文、物种、样本、assay、同步化技术、周期阶段、扰动技术、靶标、自产/复用及 accession 筛选。
3. 提供字段字典、缺失值说明、证据查看方法和常见筛选示例。
4. 生成覆盖率与问题统计，不用“完整”掩盖尚未解决的缺口。

## 八、预期产物与工程结构

建议后续在不移动原始 PDF 的前提下形成：

```text
data/
  interim/                 # 可再生成的中间提取结果
  curated/                 # 经核验的规范化表
outputs/
  papers.tsv
  experiments.tsv
  samples_timepoints.tsv
  perturbations.tsv
  accessions.tsv

  unresolved_issues.tsv
  literature_experiment_catalog.tsv  # 最终用户宽表
reports/
  per_paper/               # 每篇论文的核验摘要
  coverage_summary.md
docs/
  data_dictionary.md
  verification_protocol.md
src/                       # 解析、标准化、核验和汇总模块
```

原始 PDF 和下载数据保持只读；中间产物与最终产物分离。脚本不得硬编码本机绝对路径。大型 FASTQ 等数据默认不下载、不提交；本项目优先整理数据库元数据和官方链接，只有后续明确需要时才下载原始文件。

## 九、验证标准与验收条件

### 9.1 完整性

- 29 篇研究论文均有唯一处理记录，状态为已完成或有明确未完成原因。
- 每篇论文的所有可识别实验和数据集均有处置结果，不只整理最显眼的数据。
- 论文报告的每个公共项目号均已展开或记录无法展开的原因。
- 所有可公开解析的 Run accession 逐项列出，并与数据库返回数量完成对账。

### 9.2 准确性与可追溯性

- DOI、物种、自产数据判定、同步化、采样点、周期阶段、扰动、靶标、项目号和运行号等关键字段均有证据引用。
- ENA/EMBL-EBI 链接来自官方记录并带核验日期；链接不可访问时保留原链接和失败状态。
- 任何推导字段必须标记推导类型和规则；无法确认的信息使用规定缺失码。
- 数据库与论文冲突不被静默修正，必须进入问题表。

### 9.3 一致性与可用性

- 主外键有效，唯一键无冲突，受控字段无未定义取值。
- 最终宽表每一行可回溯到规范化表和证据表。
- 字段名稳定、说明清晰，表格可直接用常见电子表格软件打开、筛选和排序。
- 随机抽查与高风险字段人工复核后，修订记录可追踪。

## 十、当前第一阶段目标

第一轮正式任务应只推进“清单与数据标准”，不直接声称完成全部论文提取：

1. 为 29 篇研究论文建立论文清单和稳定 ID。
2. 识别疑似重复或版本文件并提出核验依据。
3. 创建规范化表头、字段字典、缺失值规范、受控词表和唯一键规则。
4. 选择 1 篇结构较清晰的论文作为试点，仅用于验证数据模型是否能容纳同步化、时间点、扰动和 accession 映射。
5. 输出试点暴露的问题和字段修订建议，等待人工确认后再批量推进。

## 十一、限制、风险与应对

- 本地仅有论文 PDF，补充材料可能缺失；必须登记缺失材料并从期刊或数据库官方入口核验。
- PDF 可能是扫描版、文本层损坏或表格解析困难；OCR/解析结果仅作提取辅助，关键字段必须回看原页。
- 部分项目可能受控访问、未到公开日期、已撤回或迁移；记录状态而不是推测内容。
- GEO、SRA、ENA、BioProject、BioSample 等编号层级容易混淆；必须保留命名空间和实体类型。
- 同一数据可能在多个数据库镜像；按 Run/Experiment/Sample 关系去重，不能只按 URL 去重。
- 论文中的样本名与数据库 alias 可能不一致；只有存在明确映射证据时才合并。
- “采样时间点对应周期阶段”常存在作者未明示的情况；不得用常识强行标注。
- “预期效果”与实验后观察结果容易混淆；分别记录并引用不同证据。
- 未来在线数据库记录可能变化；所有在线核验需保存查询日期、查询条件和必要的轻量元数据快照。
- 该项目涉及大量逐条核验，优先保证准确和可追溯，不以填满字段或追求表面完成率牺牲可信度。

## 十二、推进与停止规则

项目按 `promptn.md → 人工审查 → 明确执行 → resultn.md → 人工审查` 的交互流程推进。每轮只处理获批范围；不得自动执行新生成的 prompt，不得自动进入下一轮，不得自动提交或推送。遇到证据冲突、材料缺失、批量下载、大文件、受控数据或需要扩大研究范围时，应停止并请求用户决定。
